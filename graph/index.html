    Depth-First Search (DFS)

    Explanation:

    Depth-First Search (DFS) is a traversal method that explores as far as possible along each branch before backtracking. It uses a stack (either explicitly or via recursion).

    C++ Implementation (Recursive):

    #include <iostream>
#include <vector>

void DFSUtil(int v, std::vector<bool>& visited, const std::vector<std::vector<int>>& adj) {
    visited[v] = true;
    std::cout << v << " ";

    for (int u : adj[v]) {
        if (!visited[u]) {
            DFSUtil(u, visited, adj);
        }
    }
}

void DFS(const std::vector<std::vector<int>>& adj, int start) {
    std::vector<bool> visited(adj.size(), false);
    DFSUtil(start, visited, adj);
}

int main() {
    std::vector<std::vector<int>> adj = {
        {1, 2},
        {0, 3, 4},
        {0, 5, 6},
        {1},
        {1, 7},
        {2},
        {2},
        {4}
    };

    std::cout << "DFS starting from vertex 0:\n";
    DFS(adj, 0);

    return 0;
}

    Depth-First Search (DFS) is a fundamental algorithm used to explore nodes and edges of a graph. The primary goal of DFS is to traverse the graph, visiting each vertex and edge exactly once. It uses a stack-based approach, either through explicit use of a stack data structure or via recursion, which implicitly uses the call stack.

    Characteristics of DFS

    • Traversal Order: DFS goes as deep as possible along a branch before backtracking.
    • Applications: Used in solving puzzles, pathfinding, topological sorting, detecting cycles in graphs, and more.

    Types of Graphs

    • Directed Graph: Edges have a direction.
    • Undirected Graph: Edges do not have a direction.

    Implementation Steps

    1. Initialization: Start from a source vertex and mark it as visited.
    2. Explore: Visit an adjacent unvisited vertex, mark it as visited, and repeat the process for that vertex.
    3. Backtrack: When there are no more unvisited adjacent vertices, backtrack to the previous vertex and repeat the process.

    DFS Implementation in C++

    Here's a detailed implementation of DFS using an adjacency list representation for the graph:

    Graph Representation

    #include <iostream>
#include <vector>
#include <stack>
#include <list>

class Graph {
    int V; // Number of vertices
    std::vector<std::list<int>> adj; // Adjacency list

public:
    Graph(int V);
    void addEdge(int v, int w);
    void DFS(int v);
};

Graph::Graph(int V) {
    this->V = V;
    adj.resize(V);
}

void Graph::addEdge(int v, int w) {
    adj[v].push_back(w); // Add w to v's list.
}

void Graph::DFS(int v) {
    std::vector<bool> visited(V, false); // Mark all the vertices as not visited

    std::stack<int> stack; // Create a stack for DFS
    stack.push(v); // Push the current source node.

    while (!stack.empty()) {
        v = stack.top();
        stack.pop();

        if (!visited[v]) {
            std::cout << v << " ";
            visited[v] = true;
        }

        for (auto i = adj[v].rbegin(); i != adj[v].rend(); ++i) {
            if (!visited[*i]) {
                stack.push(*i);
            }
        }
    }
}

int main() {
    Graph g(5); // Example graph with 5 vertices

    g.addEdge(0, 1);
    g.addEdge(0, 2);
    g.addEdge(1, 2);
    g.addEdge(2, 0);
    g.addEdge(2, 3);
    g.addEdge(3, 3);
    g.addEdge(3, 4);

    std::cout << "Depth First Traversal starting from vertex 2:\n";
    g.DFS(2);

    return 0;
}

    Explanation

    1. Graph Class:

      • V: Number of vertices.
      • adj: Adjacency list to store the graph.

    2. Constructor:

      • Initializes the number of vertices and resizes the adjacency list.

    3. addEdge:

      • Adds an edge to the graph by appending to the adjacency list.

    4. DFS Function:

      • Initializes a visited vector to keep track of visited nodes.
      • Uses a stack to manage the nodes being processed.
      • Begins from the source node, marking it as visited, and explores its unvisited neighbors.

    5. Main Function:

      • Creates a graph with 5 vertices.
      • Adds edges to the graph.
      • Calls the DFS function starting from vertex 2.

    Key Points

    • Stack-Based Approach: The stack ensures nodes are processed in a LIFO order, naturally facilitating the depth-first nature of the traversal.
    • Adjacency List: Efficient for storing sparse graphs and allows O(1) access to adjacent nodes.
    • Iterative vs. Recursive: This example uses an iterative approach with an explicit stack. The recursive approach would use the system call stack.

class Solution {
public:
    void inOrderDFS(TreeNode* root, std::vector<int>& result) {

            return;
        }
        inOrderDFS(root->left, result);  // Visit left subtree
        result.push_back(root->val);     // Visit node
        inOrderDFS(root->right, result); // Visit right subtree
    }

    std::vector<int> inOrderTraversal(TreeNode* root) {
        std::vector<int> result;
        inOrderDFS(root, result);
        return result;
    }
};

        preOrderDFS(root->left, result); // Visit left subtree
        preOrderDFS(root->right, result);// Visit right subtree

        std::vector<int> result;
        preOrderDFS(root, result);

    Breadth-First Search (BFS)

    Explanation:

    Breadth-First Search (BFS) is a traversal method that explores all neighbors at the present depth level before moving on to nodes at the next depth level. It uses a queue to keep track of the nodes to be visited.

    C++ Implementation:

    #include <iostream>
#include <vector>
#include <queue>

void BFS(const std::vector<std::vector<int>>& adj, int start) {
    std::vector<bool> visited(adj.size(), false);
    std::queue<int> q;

    visited[start] = true;
    q.push(start);

    while (!q.empty()) {
        int v = q.front();
        q.pop();
        std::cout << v << " ";

        for (int u : adj[v]) {
            if (!visited[u]) {
                visited[u] = true;
                q.push(u);
            }
        }
    }
}

int main() {
    std::vector<std::vector<int>> adj = {
        {1, 2},
        {0, 3, 4},
        {0, 5, 6},
        {1},
        {1, 7},
        {2},
        {2},
        {4}
    };

    std::cout << "BFS starting from vertex 0:\n";
    BFS(adj, 0);

    return 0;
}

    Breadth-First Search (BFS) is a fundamental graph traversal algorithm used to explore the vertices and edges of a graph in layers. Unlike DFS, which goes as deep as possible along a branch before backtracking, BFS explores all neighbors at the present depth level before moving on to nodes at the next depth level. This algorithm uses a queue to manage the order of exploration.

    Characteristics of BFS

    • Traversal Order: Visits nodes level by level.
    • Applications: Used in finding the shortest path in an unweighted graph, level-order traversal in trees, peer-to-peer networks, web crawlers, etc.

    Types of Graphs

    • Directed Graph: Edges have a direction.
    • Undirected Graph: Edges do not have a direction.

    Implementation Steps

    1. Initialization: Start from a source vertex, mark it as visited, and enqueue it.
    2. Explore: Dequeue a vertex from the queue, visit all its unvisited neighbors, mark them as visited, and enqueue them.
    3. Repeat: Continue until the queue is empty.

    BFS Implementation in C++

    Here's a detailed implementation of BFS using an adjacency list representation for the graph:

    Graph Representation
    #include <iostream>
#include <vector>
#include <queue>
#include <list>

class Graph {
    int V; // Number of vertices
    std::vector<std::list<int>> adj; // Adjacency list

public:
    Graph(int V);
    void addEdge(int v, int w);
    std::vector<int> BFS(int s);
};

Graph::Graph(int V) {
    this->V = V;
    adj.resize(V);
}

void Graph::addEdge(int v, int w) {
    adj[v].push_back(w); // Add w to v's list.
}

std::vector<int> Graph::BFS(int s) {
    std::vector<bool> visited(V, false); // Mark all the vertices as not visited
    std::vector<int> bfs_order; // To store the BFS traversal order

    std::queue<int> queue; // Create a queue for BFS

    visited[s] = true; // Mark the current node as visited and enqueue it
    queue.push(s);

    while (!queue.empty()) {
        s = queue.front(); // Dequeue a vertex from queue
        queue.pop();
        bfs_order.push_back(s);

        for (auto i = adj[s].begin(); i != adj[s].end(); ++i) {
            if (!visited[*i]) {
                visited[*i] = true;
                queue.push(*i);
            }
        }
    }

    return bfs_order;
}

int main() {
    Graph g(5); // Example graph with 5 vertices

    g.addEdge(0, 1);
    g.addEdge(0, 2);
    g.addEdge(1, 2);
    g.addEdge(2, 0);
    g.addEdge(2, 3);
    g.addEdge(3, 3);
    g.addEdge(3, 4);

    std::vector<int> result = g.BFS(2);

    std::cout << "Breadth First Traversal starting from vertex 2:\n";
    for (int i : result) {
        std::cout << i << " ";
    }

    return 0;
}

    Explanation

    1. Graph Class:

      • V: Number of vertices.
      • adj: Adjacency list to store the graph.

    2. Constructor:

      • Initializes the number of vertices and resizes the adjacency list.

    3. addEdge:

      • Adds an edge to the graph by appending to the adjacency list.

    4. BFS Function:

      • Initializes a visited vector to keep track of visited nodes.
      • Uses a queue to manage the nodes being processed.
      • Begins from the source node, marking it as visited, and explores its unvisited neighbors.

    5. Main Function:

      • Creates a graph with 5 vertices.
      • Adds edges to the graph.
      • Calls the BFS function starting from vertex 2 and prints the BFS traversal order.

    Key Points

    • Queue-Based Approach: The queue ensures nodes are processed in a FIFO order, naturally facilitating the breadth-first nature of the traversal.
    • Adjacency List: Efficient for storing sparse graphs and allows O(1) access to adjacent nodes.
    • Iterative Approach: BFS is inherently iterative and relies on a queue to manage the order of exploration.

    Example with a Binary Tree

    Here's an example of performing a BFS traversal (level-order traversal) on a binary tree, which is more common in LeetCode problems.

    Definition for a binary tree node:
    struct TreeNode {
    int val;
    TreeNode *left;
    TreeNode *right;
    TreeNode(int x) : val(x), left(NULL), right(NULL) {}
};
    BFS Function Implementation for a Binary Tree
    #include <iostream>
#include <vector>
#include <queue>

class Solution {
public:
    std::vector<std::vector<int>> levelOrder(TreeNode* root) {
        std::vector<std::vector<int>> result;
        if (root == nullptr) {
            return result;
        }

        std::queue<TreeNode*> queue;
        queue.push(root);

        while (!queue.empty()) {
            int level_size = queue.size();
            std::vector<int> level;

            for (int i = 0; i < level_size; ++i) {
                TreeNode* node = queue.front();
                queue.pop();
                level.push_back(node->val);

                if (node->left != nullptr) {
                    queue.push(node->left);
                }
                if (node->right != nullptr) {
                    queue.push(node->right);
                }
            }

            result.push_back(level);
        }

        return result;
    }
};

    Iterative Deepening Depth-First Search (IDDFS)

    Explanation:

    Iterative Deepening Depth-First Search (IDDFS) combines the space efficiency of DFS with the optimality and completeness of BFS. It performs DFS with increasing depth limits.

    C++ Implementation:

    #include <iostream>
#include <vector>

bool DLS(int v, std::vector<bool>& visited, const std::vector<std::vector<int>>& adj, int depth) {
    if (depth == 0) {
        std::cout << v << " ";
        return true;
    }
    if (depth > 0) {
        visited[v] = true;
        for (int u : adj[v]) {
            if (!visited[u]) {
                if (DLS(u, visited, adj, depth - 1)) {
                    return true;
                }
            }
        }
        visited[v] = false;
    }
    return false;
}

void IDDFS(const std::vector<std::vector<int>>& adj, int start, int maxDepth) {
    for (int depth = 0; depth <= maxDepth; ++depth) {
        std::vector<bool> visited(adj.size(), false);
        if (DLS(start, visited, adj, depth)) {
            return;
        }
    }
}

int main() {
    std::vector<std::vector<int>> adj = {
        {1, 2},
        {0, 3, 4},
        {0, 5, 6},
        {1},
        {1, 7},
        {2},
        {2},
        {4}
    };

    std::cout << "IDDFS starting from vertex 0 up to depth 3:\n";
    IDDFS(adj, 0, 3);

    return 0;
}

    Shortest Path Algorithms

    Shortest path algorithms are fundamental in graph theory and computer science, used to find the shortest path between nodes in a graph. Let's explore six well-known algorithms: Dijkstra's Algorithm, Bellman-Ford Algorithm, Floyd-Warshall Algorithm, Johnson's Algorithm, A* Search Algorithm, and Bidirectional Search.

    Dijkstra's Algorithm

    Explanation:

    Dijkstra's Algorithm finds the shortest paths from a source node to all other nodes in a graph with non-negative edge weights.

    C++ Implementation:

    #include <iostream>
#include <vector>
#include <queue>
#include <limits>

const int INF = std::numeric_limits<int>::max();

void dijkstra(const std::vector<std::vector<std::pair<int, int>>>& graph, int source) {
    int V = graph.size();
    std::vector<int> dist(V, INF);
    dist[source] = 0;

    std::priority_queue<std::pair<int, int>, std::vector<std::pair<int, int>>, std::greater<>> pq;
    pq.push({0, source});

    while (!pq.empty()) {
        int u = pq.top().second;
        pq.pop();

        for (const auto& neighbor : graph[u]) {
            int v = neighbor.first;
            int weight = neighbor.second;

            if (dist[u] + weight < dist[v]) {
                dist[v] = dist[u] + weight;
                pq.push({dist[v], v});
            }
        }
    }

    std::cout << "Dijkstra's Algorithm:" << std::endl;
    for (int i = 0; i < V; i++) {
        std::cout << "Distance from source to vertex " << i << " is " << dist[i] << std::endl;
    }
}

int main() {
    std::vector<std::vector<std::pair<int, int>>> graph = {
        {{1, 4}, {2, 1}},
        {{2, 2}, {3, 1}},
        {{1, 2}, {3, 5}},
        {}
    };

    dijkstra(graph, 0);

    return 0;
}
    Dijkstra's algorithm is a famous algorithm used for finding the shortest paths between nodes in a graph, which may represent, for example, road networks. It works by iteratively selecting the vertex with the smallest known distance, updating the distances to its neighbors, and repeating until all vertices have been processed.

    Characteristics of Dijkstra's Algorithm

    • Graph Type: Works for both directed and undirected graphs with non-negative edge weights.
    • Purpose: Finds the shortest path from a source vertex to all other vertices in the graph.
    • Data Structures: Utilizes a priority queue to efficiently fetch the vertex with the smallest distance.

    Implementation Steps

    1. Initialization: Set the distance to the source vertex to 0 and to all other vertices to infinity. Insert the source vertex into the priority queue.
    2. Relaxation: Extract the vertex with the smallest distance from the priority queue, update the distances to its neighbors, and reinsert them into the queue if their distances have been updated.
    3. Repeat: Continue this process until the priority queue is empty.

    dist[src] = 0; // Distance to the source is 0

    // Min-heap priority queue to store vertices and their distances

    pq.push(std::make_pair(0, src));

                pq.push(std::make_pair(dist[v], v));
            }
        }
    }

    return dist;
}

    Floyd-Warshall Algorithm

    Explanation:

    The Floyd-Warshall Algorithm finds the shortest paths between all pairs of nodes in a graph. It works with both positive and negative edge weights but not with negative weight cycles.

    • Purpose: Finds the shortest paths between all pairs of vertices.
    • Dynamic Programming: Uses a 2D array to store shortest path estimates and updates them iteratively.

    Johnson's Algorithm

    Explanation:

    Johnson's Algorithm finds the shortest paths between all pairs of nodes in a sparse graph. It uses Bellman-Ford to reweight the graph and then applies Dijkstra's algorithm.

    C++ Implementation:

#include <vector>
#include <queue>
#include <limits>
#include <tuple>

const int INF = std::numeric_limits<int>::max();

void johnson(const std::vector<std::vector<std::pair<int, int>>>& graph) {
    int V = graph.size();
    std::vector<std::vector<int>> dist(V, std::vector<int>(V, INF));

    std::vector<std::tuple<int, int, int>> edges;
    for (int u = 0; u < V; u++) {
        for (const auto& neighbor : graph[u]) {
            int v = neighbor.first;
            int weight = neighbor.second;
            edges.push_back({u, v, weight});
        }
    }

    std::vector<int> h(V + 1, 0);
    for (int i = 1; i <= V; i++) {
        for (const auto& edge : edges) {
            int u, v, weight;
            std::tie(u, v, weight) = edge;
            if (h[u] != INF && h[u] + weight < h[v]) {
                h[v] = h[u] + weight;
            }
        }
    }

    for (auto& edge : edges) {
        int u, v, weight;
        std::tie(u, v, weight) = edge;
        weight += h[u] - h[v];
    }

    for (int u = 0; u < V; u++) {
        std::priority_queue<std::pair<int, int>, std::vector<std::pair<int, int>>, std::greater<>> pq;

        dist[u][u] = 0;

        while (!pq.empty()) {
            int d = pq.top().first;
            int v = pq.top().second;
            pq.pop();

            if (d != dist[u][v]) continue;

            for (const auto& neighbor : graph[v]) {
                int to = neighbor.first;
                int len = neighbor.second;

                if (dist[u][v] + len < dist[u][to]) {
                    dist[u][to] = dist[u][v] + len;
                    pq.push({dist[u][to], to});
                }
            }
        }
    }

        }
    }

    return dist;
}

    A* Search Algorithm

    Explanation:

    A* Search Algorithm finds the shortest path from a start node to a goal node using heuristics to guide the search.

cmath>

const int INF = std::numeric_limits<int>::max();

struct Node {
    int x, y;
    Node(int x, int y) : x(x), y(y) {}
};

double heuristic(const Node& a, const Node& b) {
    return std::sqrt((a.x - b.x) * (a.x - b.x) + (a.y - b.y) * (a.y - b.y));
}

void aStar(const std::vector<std::vector<std::pair<int, double>>>& graph, const Node& start, const Node& goal) {
    int V = graph.size();
    std::vector<double> dist(V, INF);
    std::vector<double> fScore(V, INF);
    std::vector<int> prev(V, -1);

    auto cmp = [&](int left, int right) { return fScore[left] > fScore[right]; };
    std::priority_queue<int, std::vector<int>, decltype(cmp)> openSet(cmp);

    dist[start.x] = 0;
    fScore[start.x] = heuristic(start, goal);
    openSet.push(start.x);

    while (!openSet.empty()) {
        int current = openSet.top();
        openSet.pop();

        if (current == goal.x) break;

        for (const auto& neighbor : graph[current]) {
            int next = neighbor.first;
            double weight = neighbor.second;

            double tentative_gScore = dist[current] + weight;
            if (tentative_gScore < dist[next]) {
                prev[next] = current;
                dist[next] = tentative_gScore;
                fScore[next] = dist[next] + heuristic(Node(next, 0), goal);
                openSet.push(next);
            }
        }
    }

    std::cout << "A* Search Algorithm:" << std::endl;
    if (dist[goal.x] == INF) {
        std::cout << "No path found" << std::endl;
    } else {
        std::cout << "Shortest path cost: " << dist[goal.x] << std::endl;
        std::vector<int> path;
        for (int at = goal.x; at != -1; at = prev[at]) {
            path.push_back(at);
        }
        std::reverse(path.begin(), path.end());
        std::cout << "Path: ";
        for (int p : path) {
            std::cout << p << " ";
        }
        std::cout << std::endl;
    }
}

    2. Processing: Extract the node with the lowest f from the priority queue, mark it as processed, and update the costs of its neighbors.
    3. Repeat: Continue the process until the goal node is reached or the priority queue is empty.

    Bidirectional Search

    Explanation:

    Bidirectional Search finds the shortest path between two nodes by simultaneously searching forward from the source node and backward from the target node.

    C++ Implementation:

    #include <iostream>

bool bfs(const std::vector<std::vector<int>>& graph, std::vector<int>& dist, std::queue<int>& q, int end) {
    if (q.empty()) return false;

    int node = q.front();
    q.pop();

    for (int neighbor : graph[node]) {
        if (dist[neighbor] == INF) {
            dist[neighbor] = dist[node] + 1;
            q.push(neighbor);
            if (neighbor == end) return true;
        }
    }

    return false;
}

int bidirectionalSearch(const std::vector<std::vector<int>>& graph, int start, int goal) {
    std::vector<int> distStart(graph.size(), INF);
    std::vector<int> distGoal(graph.size(), INF);

    std::queue<int> qStart, qGoal;
    qStart.push(start);
    qGoal.push(goal);

    distStart[start] = 0;
    distGoal[goal] = 0;

    while (!qStart.empty() && !qGoal.empty()) {
        if (bfs(graph, distStart, qStart, goal)) {
            break;
        }
        if (bfs(graph, distGoal, qGoal, start)) {
            break;
        }
    }

    int shortestPath = INF;
    for (int i = 0; i < graph.size(); i++) {
        if (distStart[i] != INF && distGoal[i] != INF) {
            shortestPath = std::min(shortestPath, distStart[i] + distGoal[i]);
        }
    }

    return (shortestPath == INF) ? -1 : shortestPath;
}

    Minimum Spanning Tree (MST) Algorithms

    Minimum Spanning Tree algorithms are used to find a subset of edges in a weighted graph that connects all vertices together without any cycles and with the minimum possible total edge weight.     Kruskal's Algorithm

    Explanation:

    Kruskal's Algorithm works by sorting all edges in the graph by weight and adding them one by one to the growing spanning tree, ensuring no cycles are formed, using a disjoint-set (union-find) data structure.

    C++ Implementation:

    #include <iostream>

#include <algorithm>

    int src, dest, weight;

struct DisjointSets {
    std::vector<int> parent, rank;
    DisjointSets(int n) : parent(n), rank(n, 0) {
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    int find(int u) {
        if (u != parent[u])
            parent[u] = find(parent[u]);
        return parent[u];
    }

    void merge(int x, int y) {
        int rootX = find(x);
        int rootY = find(y);

        if (rootX != rootY) {
            if (rank[rootX] > rank[rootY])
                parent[rootY] = rootX;
            else if (rank[rootX] < rank[rootY])
                parent[rootX] = rootY;
            else {
                parent[rootY] = rootX;
                rank[rootX]++;
            }
        }
    }
};

void KruskalMST(Graph& graph) {
    std::vector<Edge> result;
    std::sort(graph.edges.begin(), graph.edges.end(), [](Edge a, Edge b) {
        return a.weight < b.weight;
    });

    DisjointSets ds(graph.V);
    for (const auto& edge : graph.edges) {
        int u = edge.src;
        int v = edge.dest;
        if (ds.find(u) != ds.find(v)) {
            result.push_back(edge);
            ds.merge(u, v);
        }
    }

    std::cout << "Kruskal's MST:" << std::endl;
    for (const auto& edge : result) {
        std::cout << edge.src << " - " << edge.dest << ": " << edge.weight << std::endl;
    }
}

    Prim's Algorithm

    Explanation:

    Prim's Algorithm starts with a single vertex and grows the spanning tree one edge at a time by adding the cheapest edge from the tree to a vertex not yet in the tree.

    std::vector<int> cheapest(graph.V, -1);

    while (numTrees > 1) {
        for (int i = 0; i < graph.V; i++) {
            cheapest[i] = -1;
        }

        for (int i = 0; i < graph.E; i++) {
            int u = ds.find(graph.edges[i].src);
            int v = ds.find(graph.edges[i].dest);

            if (u == v) continue;

            if (cheapest[u] == -1 || graph.edges[cheapest[u]].weight > graph.edges[i].weight) {
                cheapest[u] = i;
            }
            if (cheapest[v] == -1 || graph.edges[cheapest[v]].weight > graph.edges[i].weight) {
                cheapest[v] = i;
            }
        }

        for (int i = 0; i < graph.V; i++) {
            if (cheapest[i] != -1) {
                int u = ds.find(graph.edges[cheapest[i]].src);
                int v = ds.find(graph.edges[cheapest[i]].dest);

                if (u != v) {
                    result.push_back(graph.edges[cheapest[i]]);
                    ds.merge(u, v);
                    numTrees--;
                }
            }
        }
    }

    std::cout << "Borůvka's MST:" << std::endl;
    for (const auto& edge : result) {
        std::cout << edge.src << " - " << edge.dest << ": " << edge.weight << std::endl;
    }
}

    BoruvkaMST(graph);

    Cycle Detection

    Floyd's Cycle-Finding Algorithm (Tortoise and Hare)

    Explanation:

    Floyd's Cycle-Finding Algorithm uses two pointers moving at different speeds (one moving twice as fast as the other). If there's a cycle, the two pointers will eventually meet inside the cycle.

    C++ Implementation:

    #include <iostream>

// Definition for singly-linked list node
struct ListNode {
    int val;
    ListNode* next;
    ListNode(int x) : val(x), next(nullptr) {}
};

bool hasCycle(ListNode* head) {
    if (!head || !head->next) return false;

    ListNode* slow = head;
    ListNode* fast = head;

    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) {
            return true;
        }
    }

    return false;
}

int main() {
    // Creating a linked list with a cycle
    ListNode* head = new ListNode(3);
    head->next = new ListNode(2);
    head->next->next = new ListNode(0);
    head->next->next->next = new ListNode(-4);
    head->next->next->next->next = head->next; // Cycle here

    if (hasCycle(head)) {
        std::cout << "Cycle detected in the linked list." << std::endl;
    } else {
        std::cout << "No cycle in the linked list." << std::endl;
    }

    return 0;
}

    Brent's Algorithm

    Explanation:

    Brent's Algorithm also uses two pointers but improves on Floyd's algorithm by using powers of two to determine the cycle length. It minimizes the number of comparisons and thus can be more efficient in some cases.

    C++ Implementation:

    #include <iostream>

// Definition for singly-linked list node
struct ListNode {
    int val;
    ListNode* next;
    ListNode(int x) : val(x), next(nullptr) {}
};

bool hasCycle(ListNode* head) {
    if (!head) return false;

    ListNode* tortoise = head;
    ListNode* hare = head;
    int power = 1;
    int lambda = 1;

    while (hare->next) {
        hare = hare->next;
        if (tortoise == hare) return true;

        if (lambda == power) {
            tortoise = hare;
            power *= 2;
            lambda = 0;
        }
        lambda++;
    }

    return false;
}

int main() {
    // Creating a linked list with a cycle
    ListNode* head = new ListNode(3);
    head->next = new ListNode(2);
    head->next->next = new ListNode(0);
    head->next->next->next = new ListNode(-4);
    head->next->next->next->next = head->next; // Cycle here

    if (hasCycle(head)) {
        std::cout << "Cycle detected in the linked list." << std::endl;
    } else {
        std::cout << "No cycle in the linked list." << std::endl;
    }

    return 0;
}

    Explanation of the Code:

    1. ListNode Structure:

      • This struct defines a singly-linked list node with an integer value and a pointer to the next node.

    2. Floyd's Cycle-Finding Algorithm:

      • Two pointers, slow and fast, are initialized to the head of the list.
      • slow moves one step at a time, while fast moves two steps.
      • If slow and fast meet, there's a cycle. If fast reaches the end of the list, there's no cycle.

    3. Brent's Algorithm:

      • Two pointers, tortoise and hare, are initialized to the head of the list.
      • hare moves one step at a time. When lambda (the length of the cycle) equals power, tortoise is moved to the current position of hare, and power is doubled.
      • If tortoise and hare meet, there's a cycle. If hare reaches the end of the list, there's no cycle.

    Both algorithms are efficient for detecting cycles in linked lists and can be used based on specific performance needs or preferences.

    Graph Coloring

    Graph coloring is the process of assigning colors to the vertices of a graph so that no two adjacent vertices share the same color. This can be used in various applications such as scheduling, register allocation in compilers, and frequency assignment in wireless networks.

    Let's explore two well-known graph coloring algorithms: the Greedy Coloring Algorithm and the Welsh-Powell Algorithm.

    Greedy Coloring Algorithm

    Explanation:

    The Greedy Coloring Algorithm assigns colors to the vertices one by one. It picks the first available color that hasn't been used by its adjacent vertices. While not always optimal, it often performs well in practice.

    C++ Implementation:

    #include <iostream>
#include <vector>
#include <algorithm>

// Function to find the minimum available color
int getFirstAvailableColor(const std::vector<bool>& available) {
    for (int color = 0; color < available.size(); ++color) {
        if (available[color]) {
            return color;
        }
    }
    return available.size(); // Should never reach here
}

void greedyColoring(const std::vector<std::vector<int>>& graph) {
    int V = graph.size();
    std::vector<int> result(V, -1); // Store the result of colors
    result[0] = 0; // Assign the first color to the first vertex

    std::vector<bool> available(V, true); // Keep track of available colors

    for (int u = 1; u < V; ++u) {
        // Mark colors of adjacent vertices as unavailable
        for (int i : graph[u]) {
            if (result[i] != -1) {
                available[result[i]] = false;
            }
        }

        // Find the first available color
        result[u] = getFirstAvailableColor(available);

        // Reset the values back to true for the next iteration
        for (int i : graph[u]) {
            if (result[i] != -1) {
                available[result[i]] = true;
            }
        }
    }

    // Print the result
    for (int u = 0; u < V; ++u) {
        std::cout << "Vertex " << u << " --->  Color " << result[u] << std::endl;
    }
}

        {0, 1, 2, 4},
        {2, 3}

}

    Welsh-Powell Algorithm

    Explanation:

    The Welsh-Powell Algorithm colors the vertices of a graph in decreasing order of their degrees (i.e., the number of edges connected to a vertex). This helps in reducing the total number of colors used.

    C++ Implementation:

    #include <iostream>
#include <vector>
#include <algorithm>

// Function to sort vertices by descending degree
std::vector<int> sortVerticesByDegree(const std::vector<std::vector<int>>& graph) {
    int V = graph.size();
    std::vector<std::pair<int, int>> degreeVertexPair(V);

    for (int i = 0; i < V; ++i) {
        degreeVertexPair[i] = {graph[i].size(), i};
    }

    std::sort(degreeVertexPair.begin(), degreeVertexPair.end(), std::greater<>());

    std::vector<int> sortedVertices(V);
    for (int i = 0; i < V; ++i) {
        sortedVertices[i] = degreeVertexPair[i].second;
    }

    return sortedVertices;
}

void welshPowellColoring(const std::vector<std::vector<int>>& graph) {
    int V = graph.size();
    std::vector<int> result(V, -1); // Store the result of colors

    std::vector<int> sortedVertices = sortVerticesByDegree(graph);

    int color = 0;
    for (int u : sortedVertices) {
        if (result[u] == -1) {
            result[u] = color;

            for (int v : sortedVertices) {
                if (result[v] == -1) {
                    bool canColor = true;
                    for (int adj : graph[v]) {
                        if (result[adj] == color) {
                            canColor = false;
                            break;
                        }
                    }
                    if (canColor) {
                        result[v] = color;
                    }
                }
            }

            color++;
        }
    }

    // Print the result
    for (int u = 0; u < V; ++u) {
        std::cout << "Vertex " << u << " --->  Color " << result[u] << std::endl;
    }
}

        {1, 2, 3},
        {0, 2, 3},
        {0, 1, 3, 4},
        {0, 1, 2, 4},
        {2, 3}
    };

    std::cout << "Welsh-Powell Coloring of the graph:" << std::endl;
    welshPowellColoring(graph);

    return 0;
}

    1. Greedy Coloring Algorithm:

      • Graph Representation: The graph is represented as an adjacency list, where graph[i] contains all the vertices adjacent to vertex i.

    Traveling Salesman Problem (TSP)

    The Traveling Salesman Problem (TSP) is a classic optimization problem in which a salesman needs to visit a set of cities exactly once and return to the starting city, with the goal of minimizing the total travel distance. Let's explore three algorithms for solving or approximating the TSP: the Held-Karp Algorithm, the Nearest Neighbor Algorithm, and the Christofides Algorithm.

    Held-Karp Algorithm (Dynamic Programming)

    Explanation:

    The Held-Karp Algorithm uses dynamic programming to solve the TSP optimally. It keeps track of the minimum cost to reach a subset of cities ending at a specific city.

    C++ Implementation:

    #include <iostream>
#include <vector>
#include <cmath>
#include <limits>

const int INF = std::numeric_limits<int>::max();

// Function to find the minimum cost using Held-Karp Algorithm
int tsp(const std::vector<std::vector<int>>& dist) {
    int n = dist.size();
    int VISITED_ALL = (1 << n) - 1;

    std::vector<std::vector<int>> dp(n, std::vector<int>(1 << n, INF));
    dp[0][1] = 0; // Starting from the first city

    for (int mask = 1; mask < (1 << n); mask++) {
        for (int u = 0; u < n; u++) {
            if (mask & (1 << u)) {
                for (int v = 0; v < n; v++) {
                    if (!(mask & (1 << v))) {
                        dp[v][mask | (1 << v)] = std::min(dp[v][mask | (1 << v)], dp[u][mask] + dist[u][v]);
                    }
                }
            }
        }
    }

    int minCost = INF;
    for (int i = 1; i < n; i++) {
        minCost = std::min(minCost, dp[i][VISITED_ALL] + dist[i][0]);
    }

    return minCost;
}

    std::vector<std::vector<int>> dist = {
        {0, 10, 15, 20},
        {10, 0, 35, 25},
        {15, 35, 0, 30},
        {20, 25, 30, 0}
    };

    std::cout << "Minimum cost using Held-Karp Algorithm: " << tsp(dist) << std::endl;

    return 0;
}

    Nearest Neighbor Algorithm (Greedy Approach)

    Explanation:

    The Nearest Neighbor Algorithm is a greedy approach that starts from an arbitrary city and repeatedly visits the nearest unvisited city until all cities have been visited.

    C++ Implementation:

    #include <iostream>

#include <algorithm>
#include <cmath>

int nearestNeighbor(const std::vector<std::vector<int>>& dist, int start) {
    int n = dist.size();
    std::vector<bool> visited(n, false);
    visited[start] = true;
    int totalCost = 0;
    int current = start;

    for (int i = 0; i < n - 1; i++) {
        int nextCity = -1;
        int minDist = std::numeric_limits<int>::max();

        for (int j = 0; j < n; j++) {
            if (!visited[j] && dist[current][j] < minDist) {
                nextCity = j;
                minDist = dist[current][j];
            }
        }

        visited[nextCity] = true;
        totalCost += minDist;
        current = nextCity;
    }

    totalCost += dist[current][start]; // Return to the starting city
    return totalCost;
}

    Christofides Algorithm (Approximation)

    Explanation:

    Christofides Algorithm is a 3/2-approximation algorithm for the TSP. It constructs a minimum spanning tree (MST), finds a minimum-weight perfect matching for the odd-degree vertices of the MST, and combines these to create an Eulerian circuit, which is then converted to a Hamiltonian circuit (the TSP solution).

#include <algorithm>
#include <limits>

// Function to find a Minimum Spanning Tree using Prim's Algorithm
std::vector<int> primMST(const std::vector<std::vector<int>>& dist) {
    int n = dist.size();
    std::vector<int> key(n, INF);
    std::vector<bool> inMST(n, false);
    std::vector<int> parent(n, -1);

    key[0] = 0;

    for (int count = 0; count < n - 1; count++) {
        int u = -1;
        int minKey = INF;
        for (int v = 0; v < n; v++) {
            if (!inMST[v] && key[v] < minKey) {
                minKey = key[v];
                u = v;
            }
        }

        inMST[u] = true;

        for (int v = 0; v < n; v++) {
            if (dist[u][v] && !inMST[v] && dist[u][v] < key[v]) {
                key[v] = dist[u][v];
                parent[v] = u;
            }
        }
    }

    return parent;
}

    Hamiltonian Path and Circuit

    Explanation: A Hamiltonian Path in a graph is a path that visits each vertex exactly once. A Hamiltonian Circuit is a Hamiltonian Path that starts and ends at the same vertex, forming a cycle.

    Backtracking Algorithm

    The backtracking approach tries all possible paths in the graph. It starts from a vertex and recursively explores all possible paths. If a path doesn't lead to a solution, it backtracks and tries a different path.

    Here's an implementation in C++:

    #include <iostream>
#include <vector>

class Graph {
    int V;
    std::vector<std::vector<bool>> adjMatrix;
    std::vector<int> path;

public:
    Graph(int vertices) : V(vertices), adjMatrix(V, std::vector<bool>(V, false)), path(V, -1) {}

    void addEdge(int v, int w) {
        adjMatrix[v][w] = true;
        adjMatrix[w][v] = true;
    }

    bool isSafe(int v, int pos) {
        if (!adjMatrix[path[pos - 1]][v])
            return false;

        for (int i = 0; i < pos; i++)
            if (path[i] == v)
                return false;

        return true;
    }

    bool hamiltonianUtil(int pos) {
        if (pos == V) {
            // Check if there's an edge from the last vertex to the first
            return adjMatrix[path[pos - 1]][path[0]];
        }

        for (int v = 1; v < V; v++) {
            if (isSafe(v, pos)) {
                path[pos] = v;

                if (hamiltonianUtil(pos + 1))
                    return true;

                path[pos] = -1;
            }
        }

        return false;
    }

    bool hamiltonian() {
        path[0] = 0;

        if (!hamiltonianUtil(1)) {
            std::cout << "No Hamiltonian Circuit exists" << std::endl;
            return false;
        }

        printSolution();
        return true;
    }

    void printSolution() {
        std::cout << "Hamiltonian Circuit exists:" << std::endl;
        for (int i = 0; i < V; i++)
            std::cout << path[i] << " ";
        std::cout << path[0] << std::endl;
    }
};

    Graph g(5);
    g.addEdge(0, 1);
    g.addEdge(0, 3);
    g.addEdge(1, 2);
    g.addEdge(1, 3);
    g.addEdge(1, 4);
    g.addEdge(2, 4);
    g.addEdge(3, 4);

    g.hamiltonian();

    return 0;
}

    Dynamic Programming Approach

    The dynamic programming approach for finding a Hamiltonian Path uses bitmasks to represent subsets of vertices. It builds solutions for larger subsets using solutions for smaller subsets.

    Here's an implementation in C++:

    #include <iostream>
#include <vector>
#include <cstring>

const int MAX_N = 15;
const int INF = 1e9;

int n, m;
int graph[MAX_N][MAX_N];
int dp[1 << MAX_N][MAX_N];
int parent[1 << MAX_N][MAX_N];

void initialize() {
    memset(graph, 0, sizeof(graph));
    memset(dp, -1, sizeof(dp));
    memset(parent, -1, sizeof(parent));
}

int tsp(int mask, int pos) {
    if (mask == (1 << n) - 1)
        return graph[pos][0];

    if (dp[mask][pos] != -1)
        return dp[mask][pos];

    int ans = INF;
    for (int city = 0; city < n; city++) {
        if ((mask & (1 << city)) == 0) {
            int newAns = graph[pos][city] + tsp(mask | (1 << city), city);
            if (newAns < ans) {
                ans = newAns;
                parent[mask][pos] = city;
            }
        }
    }

    return dp[mask][pos] = ans;
}

void printPath() {
    std::vector<int> path;
    int mask = 1, pos = 0;

    for (int i = 0; i < n; i++) {
        path.push_back(pos);
        int nextPos = parent[mask][pos];
        mask |= (1 << nextPos);
        pos = nextPos;
    }
    path.push_back(0);

    std::cout << "Hamiltonian Path: ";
    for (int city : path)
        std::cout << city << " ";
    std::cout << std::endl;
}

int main() {
    initialize();

    std::cout << "Enter number of vertices: ";
    std::cin >> n;

    std::cout << "Enter adjacency matrix:" << std::endl;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            std::cin >> graph[i][j];

    int minCost = tsp(1, 0);

    std::cout << "Minimum cost of Hamiltonian Circuit: " << minCost << std::endl;
    printPath();

    return 0;
}

    This dynamic programming solution uses the Held-Karp algorithm, which solves the Traveling Salesman Problem (TSP). The Hamiltonian Circuit is essentially a TSP where all edges have unit weight.

    Both approaches have their pros and cons:

    1. Backtracking:

      • Pros: Simple to implement, works well for small graphs.
      • Cons: Exponential time complexity, inefficient for large graphs.

    2. Dynamic Programming:

      • Pros: More efficient than backtracking, especially for larger graphs.
      • Cons: Requires more memory, implementation is more complex.

    The time complexity for the backtracking approach is O(n!), while for the dynamic programming approach it's O(n^2 * 2^n).
